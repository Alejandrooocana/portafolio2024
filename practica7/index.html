Modelo OSI

El modelo OSI (Open Systems Interconnection) es un marco conceptual que se utiliza para entender cómo funcionan las redes de computadoras. Fue desarrollado por la Organización Internacional de Normalización (ISO) en la década de 1980. El modelo OSI divide las funciones de comunicación de red en siete capas diferentes, cada una de las cuales se encarga de una parte específica del proceso de comunicación. Aquí tienes una breve descripción de cada capa:

1. Capa física: Esta capa se encarga de la transmisión física de los datos a través del medio de comunicación, como cables de cobre, fibra óptica o ondas de radio. Define las características eléctricas, mecánicas y funcionales de los dispositivos de red.

2. Capa de enlace de datos: Esta capa se encarga de la transferencia fiable de datos a través de un enlace físico. Divide los datos en tramas y añade información de control para asegurar que los datos se transmitan sin errores.

3. Capa de red: Esta capa se encarga de la determinación de la ruta que deben seguir los datos a través de la red. También se encarga de la gestión del tráfico y de la prevención de congestiones.

4. Capa de transporte: Esta capa se encarga de la segmentación y el transporte de los datos entre sistemas finales. Proporciona servicios de control de flujo y de control de errores.

5. Capa de sesión: Esta capa establece, gestiona y finaliza las conexiones entre las aplicaciones de los sistemas finales. También se encarga de la sincronización y del control del diálogo entre las aplicaciones.

6. Capa de presentación: Esta capa se encarga de la representación de los datos, asegurando que los datos sean comprensibles para las aplicaciones. También se encarga de la compresión y el cifrado de los datos.

7. Capa de aplicación: Esta es la capa más cercana al usuario y se encarga de proporcionar servicios de red a las aplicaciones del usuario, como el correo electrónico, la transferencia de archivos y la navegación web.

El modelo OSI es útil porque proporciona un marco de referencia común para entender cómo funcionan las redes de computadoras, lo que facilita la comunicación entre los diferentes dispositivos y sistemas de red.

Modelo TCP/IP

El modelo TCP/IP (Transmission Control Protocol/Internet Protocol) es otro marco conceptual utilizado para describir cómo funcionan las redes de computadoras, especialmente en el contexto de Internet. A diferencia del modelo OSI, el modelo TCP/IP consta de cuatro capas en lugar de siete. Aquí tienes una descripción de cada capa:

1. Capa de Acceso a la Red (o de Enlace): Esta capa es similar a la capa de enlace de datos en el modelo OSI. Se encarga de la transmisión de datos a través del medio físico y de la detección de errores en la transmisión.

2. Capa de Internet: Esta capa es similar a la capa de red en el modelo OSI. Se encarga de la transmisión de datos a través de la red, incluida la determinación de la ruta más eficiente para enviar los datos.

3. Capa de Transporte: Esta capa es similar a la capa de transporte en el modelo OSI. Se encarga de la segmentación, el control de flujo y la corrección de errores en la transmisión de datos.

4. Capa de Aplicación: Esta capa es similar a la capa de aplicación en el modelo OSI, pero también incorpora funciones de las capas de sesión y presentación. Se encarga de proporcionar servicios de red a las aplicaciones del usuario, como el correo electrónico, la transferencia de archivos y la navegación web.

El modelo TCP/IP es el modelo más utilizado en la actualidad, especialmente en Internet, porque simplifica la arquitectura de red al combinar algunas funciones de las capas del modelo OSI. Muchos protocolos de red, como TCP, IP, HTTP y FTP, se basan en el modelo TCP/IP.

Ejemplos de aplicación

Modelo OSI:

1. Capa física: Transmitir datos a través de un cable Ethernet.

2. Capa de enlace de datos: Envío de tramas Ethernet entre dos dispositivos en una red local.

3. Capa de red: Enrutamiento de paquetes IP a través de una red de área amplia (WAN).

4. Capa de transporte: Utilización del protocolo TCP para asegurar la entrega confiable de datos.

5. Capa de sesión: Establecimiento de una sesión de videoconferencia entre dos usuarios.

6. Capa de presentación: Conversión de datos en formato ASCII a formato Unicode para mostrar en un navegador web.

7. Capa de aplicación: Uso de un navegador web para acceder a un sitio web.

Modelo TCP/IP:

1. Capa de Acceso a la Red: Transmisión de datos a través de una red Wi-Fi.

2. Capa de Internet: Envío de paquetes IP a través de Internet.

3. Capa de Transporte: Transferencia de archivos utilizando el protocolo TCP.

4. Capa de Aplicación: Acceso a un sitio web utilizando el protocolo HTTP.

- Conclusiones -

Finalmente, podemos concluir que es muy importante realizar este tipo de prácticas de investigación para poder entender estos modelos en los que se basan nuestras redes de comunicación actuales, de esta manera podemos tener una mejor comprensión sobre los elementos y los usos que le daremos al implementarlos.
